<objective>
Fix tw.update() regression: revert from task import to task modify with tag and annotation diff support

Purpose: Phase 03-02 switched tw.update() from `task modify` to `task import` to handle tags/annotations atomically. This breaks LWW sync invariants: task import drops caldavuid UDA in Docker, causing perpetual re-sync (9/18 integration tests fail). The fix reverts to task modify for scalar fields, adds +tag/-tag syntax for tags, and adds annotate/denotate commands for annotations.

Output: Working tw.update() that handles all fields without corrupting LWW timestamps. All 167 unit + 18 integration tests green.
</objective>
