    **Step 3 — Fix clippy warnings (47 total, fix in this order):**

    a) **result_large_err (21 occurrences):** Add a crate-level allow attribute to `src/lib.rs`:
    ```rust
    #![allow(clippy::result_large_err)]
    ```
    This is the correct approach because the CaldaWarriorError enum intentionally carries context strings for error reporting (decided in Phase 01: AUDIT-03). Refactoring the error enum to use Box is outside this phase's scope.

    b) **collapsible_if (6 occurrences):** In each flagged location across `src/config.rs`, `src/sync/writeback.rs`, `src/sync/mod.rs`, `src/ical.rs` — combine nested `if` statements into a single `if` with `&&`. For example:
    ```rust
    // Before:
    if condition_a {
        if condition_b {
            body
        }
    }
    // After:
    if condition_a && condition_b {
        body
    }
    ```

    c) **ptr_arg (3 occurrences: 1x `&Vec`, 2x `&mut Vec`):** Change function signatures:
    - `&Vec<T>` -> `&[T]`
    - `&mut Vec<T>` -> `&mut [T]` (only if the function doesn't need push/pop; if it does, keep `&mut Vec<T>` and add `#[allow(clippy::ptr_arg)]` on that function)

    d) **redundant_closure (2 occurrences):** Replace `.map(|x| foo(x))` with `.map(foo)` or `.map_err(|e| bar(e))` with `.map_err(bar)`.

    e) **stripping a suffix manually (2 occurrences):** Replace manual suffix stripping with `str::strip_suffix()`.

    f) **map_or can be simplified (2 occurrences):** Replace `.map_or(default, |v| expr)` with the simpler form clippy suggests (usually `.map(|v| expr).unwrap_or(default)` or a direct match).

    g) **doc list item without indentation (2 occurrences):** Fix doc comment indentation to satisfy rustdoc lint.

    h) **too_many_arguments (2 occurrences):** Add `#[allow(clippy::too_many_arguments)]` on the two flagged functions. Refactoring their signatures is outside this phase's scope.

    i) **Default implementations for MockTaskRunner and MockCalDavClient (2 occurrences):** Add `impl Default for MockTaskRunner` and `impl Default for MockCalDavClient` that delegate to `Self::new()` (or equivalent constructor).

    j) **as_ref().map() on Option (1 occurrence):** Replace `.as_ref().map(|v| v.as_slice())` with `.as_deref()`.

    k) **calls to push immediately after creation (1 occurrence):** Replace `let mut v = Vec::new(); v.push(x);` with `let v = vec![x];`.

    l) **very complex type (1 occurrence):** Add a type alias for the complex type, or add `#[allow(clippy::type_complexity)]` on the function.

    m) **large size difference between variants (1 occurrence):** This is on the error enum — already covered by the `result_large_err` allow, but if clippy still flags it separately, add `#[allow(clippy::large_enum_variant)]` on the enum definition in `src/error.rs`.

    n) **extend instead of append (1 occurrence):** Replace `.extend(other_vec.drain(..))` or `.extend(other_vec.into_iter())` with `.append(&mut other_vec)`.

    **Step 4 — Verify all fixes:**
    Run `cargo fmt --check` (must exit 0), `cargo clippy -- -D warnings` (must exit 0), `cargo test --lib` (all tests pass), `cargo test --test integration` (all tests pass).

    After creating, run `cargo deny check` to verify. If the `ring` license hash `0xbd0eed23` is wrong for the current ring version in Cargo.lock, cargo-deny will output the correct hash -- update deny.toml with the correct hash and re-run.

    **Step 2 — Create `.github/workflows/ci.yml`** (create `.github/workflows/` directory first):

    ```yaml
    name: CI

    on:
      push:
      pull_request:

    env:
      CARGO_TERM_COLOR: always

    jobs:
      lint:
        name: Lint (fmt + clippy)
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4
          - uses: dtolnay/rust-toolchain@stable
            with:
              components: rustfmt, clippy
          - uses: Swatinem/rust-cache@v2
          - name: Check formatting
            run: cargo fmt --check
          - name: Run clippy
            run: cargo clippy -- -D warnings

      test:
        name: Unit + Integration Tests
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4
          - uses: dtolnay/rust-toolchain@stable
          - uses: Swatinem/rust-cache@v2
          - name: Run unit tests
            run: cargo test --lib
          - name: Run integration tests
            run: cargo test --test integration

      e2e:
        name: E2E (Robot Framework)
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4
          - name: Create results directory
            run: mkdir -p tests/robot/results
          - name: Run RF E2E tests
            run: |
              CURRENT_UID=$(id -u) CURRENT_GID=$(id -g) \
                docker compose -f tests/robot/docker-compose.yml run --rm robot
          - name: Upload RF results on failure
            if: failure()
            uses: actions/upload-artifact@v4
            with:
              name: robot-results
              path: tests/robot/results/

      audit:
        name: Security Audit (cargo-deny)
        runs-on: ubuntu-latest
        steps:
          - uses: actions/checkout@v4
          - uses: EmbarkStudios/cargo-deny-action@v2
    ```

    Key points:
    - 4 parallel jobs: lint, test, e2e, audit (per user decision)
    - Triggered on every push and PR (per user decision)
    - RF E2E uses Docker Compose directly on ubuntu-latest runner (per user decision)
    - RF E2E uses `CURRENT_UID=$(id -u) CURRENT_GID=$(id -g)` for UID/GID passthrough (per established pattern in Makefile)
    - RF results uploaded as artifact on failure for debugging
    - Swatinem/rust-cache@v2 for cargo caching (per user decision)
    - No `actions-rs` actions (deprecated, per research)
